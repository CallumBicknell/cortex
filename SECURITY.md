# Security Policy

## Supported Versions

We release patches for security vulnerabilities. Which versions are eligible
for receiving such patches depends on the version you are using.

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1.0 | :x:                |

## Reporting a Vulnerability

Please report suspected security vulnerabilities via
[GitHub Security Advisories](https://github.com/your-org/your-repo/security/advisories)
or by emailing security@yourdomain.com.

We will acknowledge receipt of your report within 48 hours and will
provide a detailed response within 7 days indicating the next steps in
handling your report.

After the initial reply, we will keep you informed of the progress
towards a fix and full announcement, and may ask for additional
information or guidance as part of the investigation.

Please include the following information in your report:

- Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting, etc.)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected issue (line number, file name, and file path)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Any noteworthy logs or crash dumps
- If applicable, the affected version and the version that you suspect
  introduced the vulnerability
- If you have created a proof of concept, please include it

We ask that you do not disclose the issue publicly until we have had
an opportunity to address it.

## Preferred Languages

We prefer all communications to be in English.
